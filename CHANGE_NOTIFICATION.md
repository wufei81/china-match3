# 变更通知模板

**用途**: 当 PRD、API、设计等发生变更时，使用此模板通知相关方

---

## 模板 1: API 变更通知

```markdown
## [高优先级] API Schema 变更通知

**变更者**: [角色，如：Architect]  
**时间**: [YYYY-MM-DD HH:mm]  
**影响范围**: [如：DevEngineer, QAEngineer, Frontend]

### 变更内容
```diff
# 变更字段说明
- 旧字段：类型 (说明)
+ 新字段：类型 (说明)
```

### 变更原因
[说明为什么变更]

### 影响评估
| 角色 | 影响内容 | 需要行动 | 截止时间 |
|------|---------|---------|---------|
| DevEngineer | 修改实现代码 | 更新代码 | 4h |
| QAEngineer | 更新测试用例 | 修改测试 | 4h |
| Frontend | 更新 API 调用 | 修改调用 | 8h |

### 确认要求
请相关方在 **4 小时** 内回复：
- `收到` - 已了解变更
- `有疑问` - 需要讨论
- `需要延期` - 无法在截止时间内完成

[查看详情链接] [发起讨论]
```

## 模板 2: PRD 变更通知

```markdown
## [高优先级] PRD 变更通知

**变更者**: ProductManager  
**时间**: [YYYY-MM-DD HH:mm]  
**版本**: v1.2 → v1.3

### 变更摘要
- 新增：[新增内容]
- 修改：[修改内容]
- 删除：[删除内容]

### 变更详情
```diff
# 新增功能
+ 用户注册流程 (P0)

# 修改功能
~ 登录方式：邮箱 → 用户名

# 删除功能
- 第三方登录 (延期到 v2.0)
```

### 影响评估
| 角色 | 影响内容 | 需要行动 | 截止时间 |
|------|---------|---------|---------|
| Architect | API 设计变更 | 更新 API Schema | 24h |
| UI/UX Designer | 注册页面设计 | 新增设计稿 | 24h |
| DevEngineer | 后端逻辑变更 | 修改实现 | 48h |
| QAEngineer | 测试用例变更 | 更新测试 | 24h |

### 确认要求
请相关方在 **24 小时** 内回复：
- `收到` - 已了解变更
- `有疑问` - 需要讨论
- `需要延期` - 无法在截止时间内完成

[查看 PRD v1.3](docs/PRD_v1.3.md) [发起讨论]
```

## 模板 4: 测试失败通知

```markdown
## [紧急] 测试失败通知

**发现者**: QAEngineer  
**时间**: [YYYY-MM-DD HH:mm]  
**严重性**: 🔴 紧急

### 失败详情
- **测试用例**: [用例名称]
- **错误信息**: [错误描述]
- **复现步骤**: [步骤 1, 2, 3]

### 影响评估
- 阻塞开发：是/否
- 影响上线：是/否
- 需要立即修复：是/否

### 需要行动
| 角色 | 行动 | 截止时间 |
|------|------|---------|
| DevEngineer | 修复 bug | 立即 |
| ProjectManager | 评估影响 | 1h |

### 回复要求
请 DevEngineer **立即** 响应：
- `收到，立即修复`
- `需要讨论`
- `不是 bug，是特性` (开玩笑的，请认真回复)

[查看测试报告](docs/TEST_REPORT.md)
```

## 通知规则汇总

| 变更类型 | 通知对象 | 响应时间 | 优先级 |
|---------|---------|---------|-------|
| PRD 变更 | Arch, UI, Dev, QA | 24h | 高 |
| API 变更 | Dev, QA | 4h | 高 |
| 设计变更 | Dev, QA | 24h | 中 |
| 测试失败 | Dev, PM | 立即 | 紧急 |
| 阻塞发现 | PM, Tech | 1h | 紧急 |

---

## 使用说明

1. **选择模板**: 根据变更类型选择对应模板
2. **填写内容**: 替换方括号中的内容
3. **发送通知**: 通过 STATE.md 或消息系统发送
4. **追踪确认**: 在 STATE.md 中更新确认状态
5. **升级处理**: 超时未确认，升级通知
