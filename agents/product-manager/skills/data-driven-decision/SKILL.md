## 🎯 技能概述

为产品经理智能体提供数据驱动决策能力，包括 A/B 测试设计、核心指标定义、漏斗分析框架。帮助从"直觉决策"升级为"数据决策"。

---

## 📊 核心功能

### 1. A/B 测试设计能力

**目的**: 用科学方法验证产品假设，降低决策风险

#### A/B 测试流程

```
假设 → 实验设计 → 样本计算 → 执行 → 分析 → 决策
```

#### A/B 测试设计模板

```markdown
## A/B 测试方案 - [测试名称]

### 一、测试假设

- **假设**: [如果做 X，那么 Y 会提升 Z%]
- **依据**: [数据/用户反馈/行业基准]
- **置信度**: 高/中/低

### 二、实验设计

| 维度         | 对照组 (A) | 实验组 (B) |
| ------------ | ---------- | ---------- |
| **变量**     | [当前方案] | [新方案]   |
| **改动点**   | -          | [具体改动] |
| **预期影响** | -          | [预期提升] |

### 三、核心指标

| 指标类型     | 指标名称           | 当前值 | 目标提升 |
| ------------ | ------------------ | ------ | -------- |
| **核心指标** | [Primary Metric]   | X%     | +Y%      |
| **护栏指标** | [Guardrail Metric] | X%     | 不下降   |
| **辅助指标** | [Secondary Metric] | X%     | -        |

### 四、样本量计算

- **当前转化率**: X%
- **预期提升**: Y%
- **统计功效**: 80%
- **显著性水平**: 95%
- **所需样本**: 每组 N 人
- **预计时长**: N 天 (基于日活)

### 五、分流策略

- **分流方式**: 随机分流 (用户 ID hash)
- **分流比例**: 50% / 50%
- **分层因素**: [新用户/老用户/地区等]

### 六、成功标准

- **统计显著**: p-value < 0.05
- **实际显著**: 提升 > 最小有意义提升 (MDE)
- **护栏指标**: 无负面影响

### 七、决策规则

| 结果     | 决策              |
| -------- | ----------------- |
| 显著正向 | 全量发布          |
| 显著负向 | 放弃方案          |
| 不显著   | 延长测试/重新设计 |
```

#### 样本量估算参考

| 当前转化率 | 预期提升 | 每组样本量 (α=0.05, β=0.2) |
| ---------- | -------- | -------------------------- |
| 1%         | +10%     | ~300,000                   |
| 1%         | +20%     | ~75,000                    |
| 5%         | +10%     | ~60,000                    |
| 5%         | +20%     | ~15,000                    |
| 10%        | +10%     | ~30,000                    |
| 10%        | +20%     | ~8,000                     |

#### 常见 A/B 测试陷阱

| 陷阱           | 描述               | 避免方法         |
| -------------- | ------------------ | ---------------- |
| **提前停止**   | 未达样本量就下结论 | 预先确定样本量   |
| **多重检验**   | 同时看太多指标     | 预先确定核心指标 |
| **辛普森悖论** | 分组数据与整体相反 | 检查分层一致性   |
| **新奇效应**   | 用户因新鲜感点击   | 延长测试时间     |
| **选择偏差**   | 样本不具代表性     | 随机分流         |

### 2. 核心指标定义 (North Star Metric)

**North Star Metric (NSM)** - 指引产品方向的核心指标

#### 指标层级框架

```
North Star Metric (1 个)
    ↓
Level 1: 关键结果指标 (3-5 个)
    ↓
Level 2: 过程指标 (10-15 个)
    ↓
Level 3: 执行指标 (若干)
```

#### 指标定义模板

```markdown
## 产品指标体系 - [产品名称]

### North Star Metric

| 维度             | 定义                 |
| ---------------- | -------------------- |
| **指标名称**     | [NSM 名称]           |
| **定义**         | [精确定义]           |
| **计算公式**     | [公式]               |
| **为什么是 NSM** | [与核心价值直接相关] |
| **当前值**       | [数值]               |
| **目标值**       | [X 个月内达到 Y]     |

### Level 1: 关键结果指标

| 指标     | 定义 | 当前值 | 目标 | 负责人 |
| -------- | ---- | ------ | ---- | ------ |
| [指标 1] |      |        |      |        |
| [指标 2] |      |        |      |        |
| [指标 3] |      |        |      |        |

### Level 2: 过程指标

| 指标             | 定义 | 当前值 | 目标 |
| ---------------- | ---- | ------ | ---- |
| **获客**         |      |        |      |
| - 新增用户       |      |        |      |
| - 获客成本 (CAC) |      |        |      |
| **激活**         |      |        |      |
| - 激活率         |      |        |      |
| - 时间到激活     |      |        |      |
| **留存**         |      |        |      |
| - 次日留存       |      |        |      |
| - 7 日留存       |      |        |      |
| - 30 日留存      |      |        |      |
| **收入**         |      |        |      |
| - 付费转化率     |      |        |      |
| - ARPU           |      |        |      |
| - LTV            |      |        |      |
| **推荐**         |      |        |      |
| - NPS            |      |        |      |
| - 病毒系数 (K)   |      |        |      |

### 指标字典

| 指标     | 精确定义     | 数据来源  | 更新频率 |
| -------- | ------------ | --------- | -------- |
| [指标名] | [无歧义定义] | [表/事件] | 日/周/月 |
```

#### 好指标的特征 (HEART + AARRR)

| 框架      | 维度     | 示例指标            |
| --------- | -------- | ------------------- |
| **HEART** | 愉悦度   | NPS, 满意度         |
|           | 参与度   | DAU/MAU, 使用时长   |
|           | 接受度   | 功能采用率          |
|           | 留存率   | 次日/7 日/30 日留存 |
|           | 任务成功 | 完成率，错误率      |
| **AARRR** | 获客     | 新增用户，CAC       |
|           | 激活     | 激活率，时间到价值  |
|           | 留存     | 留存率，流失率      |
|           | 收入     | ARPU, LTV, MRR      |
|           | 推荐     | NPS, K 因子         |

### 3. 漏斗分析框架

**目的**: 识别用户流失环节，优化转化路径

#### 漏斗分析模板

```markdown
## 漏斗分析 - [转化路径名称]

### 漏斗定义

[描述用户从起点到终点的完整路径]

### 漏斗数据

| 步骤 | 行为     | 人数   | 转化率 | 流失率 |
| ---- | -------- | ------ | ------ | ------ |
| 1    | [入口]   | 10,000 | 100%   | -      |
| 2    | [行为 1] | 8,000  | 80%    | 20%    |
| 3    | [行为 2] | 5,000  | 62.5%  | 37.5%  |
| 4    | [行为 3] | 3,000  | 60%    | 40%    |
| 5    | [转化]   | 2,500  | 83.3%  | 16.7%  |

### 关键发现

- **最大流失点**: 步骤 X (流失 Y%)
- **转化瓶颈**: [描述问题]
- **优化机会**: [具体建议]

### 细分分析

| 维度         | 细分     | 整体转化率 |
| ------------ | -------- | ---------- |
| **渠道**     | 自然搜索 | X%         |
|              | 付费广告 | Y%         |
|              | 社交推荐 | Z%         |
| **设备**     | 移动端   | X%         |
|              | 桌面端   | Y%         |
| **用户类型** | 新用户   | X%         |
|              | 老用户   | Y%         |

### 优化建议

| 优先级 | 优化点         | 预期提升 | 实施成本 |
| ------ | -------------- | -------- | -------- |
| P0     | [高流失点优化] | +X%      | 低/中/高 |
| P1     | [次要优化点]   | +Y%      | 低/中/高 |
```

#### 漏斗分析最佳实践

| 实践         | 说明                                |
| ------------ | ----------------------------------- |
| **定义清晰** | 每步有明确的开始和结束事件          |
| **时间窗口** | 设置合理的转化时间窗口 (如 24 小时) |
| **排除异常** | 排除测试用户、内部流量              |
| **趋势分析** | 对比历史数据，识别异常波动          |
| **细分洞察** | 按渠道/设备/用户类型细分            |

## 🔧 使用示例

### 示例 1: 设计 A/B 测试

**输入**: "帮我设计一个注册流程优化的 A/B 测试"

**输出**:

```markdown
## A/B 测试方案 - 注册流程优化

### 测试假设

- **假设**: 如果将注册步骤从 5 步减少到 3 步，注册完成率会提升 20%
- **依据**: 当前 5 步流程流失率 60%，行业基准 3 步流程流失率 40%
- **置信度**: 中

### 实验设计

| 维度     | 对照组 (A) | 实验组 (B)             |
| -------- | ---------- | ---------------------- |
| **流程** | 5 步注册   | 3 步注册               |
| **改动** | -          | 合并信息填写，延后验证 |

### 核心指标

| 指标类型 | 指标                | 当前值 | 目标       |
| -------- | ------------------- | ------ | ---------- |
| **核心** | 注册完成率          | 40%    | +20% → 48% |
| **护栏** | 用户质量 (7 日留存) | 30%    | 不下降     |

### 样本量

- 当前转化率: 40%
- 预期提升: 20% (相对)
- 所需样本: 每组 ~4,000 人
- 预计时长: 7 天 (基于日活 1,500) ...
```

### 示例 2: 定义指标体系

**输入**: "帮企业 IM 产品定义指标体系"

**输出**:

```markdown
## 指标体系 - 企业 IM

### North Star Metric

| 维度       | 定义                                                         |
| ---------- | ------------------------------------------------------------ |
| **名称**   | 周活跃团队数                                                 |
| **定义**   | 每周至少有 3 人发送消息的团队数                              |
| **公式**   | COUNT(DISTINCT team_id WHERE messages >= 3 AND members >= 3) |
| **为什么** | 直接反映产品核心价值 (团队协作)                              |

### Level 1 指标

| 指标              | 当前值 | 目标 (Q2) |
| ----------------- | ------ | --------- |
| 周活跃团队数      | 5,000  | 8,000     |
| 团队留存率 (4 周) | 60%    | 70%       |
| 付费转化率        | 5%     | 8%        |

...
```

## 📋 输出规范

1. **指标可计算**: 每个指标有明确公式和数据来源
2. **假设可验证**: A/B 测试假设必须可被数据证伪
3. **结果可行动**: 分析结论需导向具体行动
4. **统计严谨**: 遵循统计学原则 (显著性/功效)

---

## ⚠️ 注意事项

- ⛔ 避免虚荣指标 (如累计用户数)
- ⛔ A/B 测试不要同时改多个变量
- ⛔ 漏斗分析注意时间窗口一致性
- ✅ 核心指标不超过 1 个 NSM + 5 个 Level 1
