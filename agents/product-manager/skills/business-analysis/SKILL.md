## 🎯 技能概述

为产品经理智能体提供商业分析能力，包括市场规模估算、投资回报评估、竞争壁垒分析。帮助从"功能思维"升级为"商业思维"。

---

## 📊 核心功能

### 1. TAM/SAM/SOM 计算

**TAM (Total Addressable Market)** - 总可服务市场 **SAM (Serviceable Addressable Market)** - 可服务市场 **SOM
(Serviceable Obtainable Market)** - 可获得市场

#### 使用方法

```markdown
## 市场规模估算

### TAM (总可服务市场)

- **定义**: [市场总规模]
- **计算方法**:
  - 自上而下：[行业报告数据]
  - 自下而上：[目标用户数 × 单价]
- **估算值**: ¥XXX 亿/年

### SAM (可服务市场)

- **定义**: [你的产品能覆盖的市场]
- **限制因素**: [地域/渠道/产品类型]
- **估算值**: ¥XXX 亿/年 (TAM 的 X%)

### SOM (可获得市场)

- **定义**: [3-5 年内可获取的市场份额]
- **竞争因素**: [现有竞争者/进入壁垒]
- **估算值**: ¥XXX 亿/年 (SAM 的 X%)
```

#### 计算模板

| 层级 | 计算公式                  | 数据来源        | 估算值  |
| ---- | ------------------------- | --------------- | ------- |
| TAM  | 目标用户总数 × 年付费意愿 | 行业报告/统计局 | ¥\_\_\_ |
| SAM  | TAM × 可触达比例          | 渠道覆盖率      | ¥\_\_\_ |
| SOM  | SAM × 预期市场份额        | 竞争分析        | ¥\_\_\_ |

### 2. ROI 评估框架

**ROI (Return on Investment)** - 投资回报率

#### ROI 计算公式

```
ROI = (收益 - 成本) / 成本 × 100%

或

ROI = 净收益 / 投资总额 × 100%
```

#### 评估维度

| 维度         | 指标     | 计算方法                    |
| ------------ | -------- | --------------------------- |
| **收入增长** | 新增收入 | 新功能带来的付费用户 × ARPU |
| **成本节约** | 节省成本 | 自动化减少的人力/时间成本   |
| **效率提升** | 时间价值 | 节省时间 × 人力成本/小时    |
| **风险降低** | 避免损失 | 潜在风险事件概率 × 损失金额 |

#### ROI 评估模板

```markdown
## ROI 分析

### 投资成本

- 开发成本：¥\_\_\_ (人天 × 日薪)
- 运营成本：¥\_\_\_/月
- 营销成本：¥\_\_\_
- **总成本**: ¥\_\_\_

### 预期收益 (12 个月)

- 收入增长：¥\_\_\_
- 成本节约：¥\_\_\_
- **总收益**: ¥\_\_\_

### ROI 计算

- **净收益**: ¥\_\_\_ (收益 - 成本)
- **ROI**: \_\_\_% (净收益/成本)
- **回本周期**: \_\_\_ 个月

### 敏感性分析

| 场景 | 假设          | ROI     |
| ---- | ------------- | ------- |
| 乐观 | 用户增长 +30% | \_\_\_% |
| 基准 | 预期情况      | \_\_\_% |
| 悲观 | 用户增长 -30% | \_\_\_% |
```

### 3. 竞争壁垒分析

**竞争壁垒 (Moat)** - 保护竞争优势的护城河

#### 壁垒类型

| 壁垒类型     | 描述             | 强度     | 案例       |
| ------------ | ---------------- | -------- | ---------- |
| **网络效应** | 用户越多价值越大 | 🔴 强    | 微信/Slack |
| **转换成本** | 用户迁移成本高   | 🟡 中    | 企业 SaaS  |
| **品牌优势** | 用户心智占领     | 🟡 中    | 苹果/耐克  |
| **技术壁垒** | 专利/专有技术    | 🟢 弱-中 | 芯片/算法  |
| **规模效应** | 成本随规模降低   | 🟡 中    | 电商/物流  |
| **数据壁垒** | 数据积累优势     | 🟡 中    | 推荐系统   |

#### 壁垒分析模板

```markdown
## 竞争壁垒分析

### 现有壁垒评估

| 壁垒类型 | 是否有 | 强度 (1-5) | 说明 |
| -------- | ------ | ---------- | ---- |
| 网络效应 | ✅/❌  | \_/5       |      |
| 转换成本 | ✅/❌  | \_/5       |      |
| 品牌优势 | ✅/❌  | \_/5       |      |
| 技术壁垒 | ✅/❌  | \_/5       |      |
| 规模效应 | ✅/❌  | \_/5       |      |
| 数据壁垒 | ✅/❌  | \_/5       |      |

### 竞争格局

| 竞争者   | 市场份额 | 核心优势 | 弱点 |
| -------- | -------- | -------- | ---- |
| 竞争者 A | \_\_%    |          |      |
| 竞争者 B | \_\_%    |          |      |
| 我们     | \_\_%    |          |      |

### 壁垒构建计划

- **短期 (0-6 月)**: [快速建立的壁垒]
- **中期 (6-18 月)**: [需要投入的壁垒]
- **长期 (18 月+)**: [战略性壁垒]
```

## 🔧 使用示例

### 示例 1: 新项目商业可行性分析

**输入**: "帮我分析企业 IM 系统的商业可行性"

**输出**:

```markdown
## 商业分析 - 企业 IM 系统

### 市场规模 (TAM/SAM/SOM)

- **TAM**: 中国企业通信市场 ¥2,000 亿/年
- **SAM**: 中小企业 IM 市场 ¥500 亿/年 (25%)
- **SOM**: 3 年内可获取 ¥25 亿/年 (SAM 的 5%)

### ROI 预测 (3 年)

- **总投资**: ¥1,500 万
- **预期收益**: ¥4,500 万
- **ROI**: 200%
- **回本周期**: 18 个月

### 竞争壁垒

- **网络效应**: 🔴 强 (团队内部粘性)
- **转换成本**: 🟡 中 (数据迁移成本)
- **技术壁垒**: 🟢 弱 (同质化严重)
- **建议**: 强化网络效应 + 数据壁垒
```

## 📋 输出规范

1. **数据驱动**: 所有估算需标注数据来源或假设
2. **敏感性分析**: 必须包含乐观/基准/悲观三场景
3. **时间维度**: 明确分析周期 (通常 12-36 个月)
4. **可验证**: 关键假设可被后续数据验证

---

## ⚠️ 注意事项

- ⛔ 避免过度乐观估计 (默认打 7 折)
- ⛔ 必须注明关键假设
- ⛔ 区分事实数据和推测数据
- ✅ 提供数据来源或调研方法建议
