## 🎯 技能概述

为产品经理智能体提供用户研究能力，包括深度访谈提纲生成、用户画像构建、JTBD 分析。帮助从"需求接收者"升级为"用户洞察者"。

---

## 📊 核心功能

### 1. 用户访谈提纲生成

**目的**: 发现用户未表达的隐性需求，验证产品假设

#### 访谈类型

| 类型           | 阶段        | 目标           | 时长     |
| -------------- | ----------- | -------------- | -------- |
| **探索性访谈** | 产品前/早期 | 发现痛点和需求 | 45-60min |
| **验证性访谈** | 原型/开发中 | 验证解决方案   | 30-45min |
| **可用性测试** | 上线前      | 发现使用问题   | 30-60min |
| **满意度访谈** | 上线后      | 收集反馈和 NPS | 20-30min |

#### 访谈提纲模板

```markdown
## 用户访谈提纲 - [产品名称/功能]

### 一、暖场 (5 分钟)

- 自我介绍和访谈目的说明
- 建立信任，说明保密原则
- 询问基本信息 (职业/使用场景等)

### 二、背景了解 (10 分钟)

- "能描述一下你平时是如何 [做某事] 的吗？"
- "目前使用什么工具/方法解决这个问题？"
- "这个过程中最让你头疼的是什么？"

### 三、核心问题 (20-30 分钟)

#### 痛点探索

- "能具体说说上次遇到 [问题] 的情景吗？"
- "当时你是怎么解决的？"
- "如果不解决会有什么后果？"

#### 需求挖掘

- "你理想中的解决方案是什么样的？"
- "如果有一个魔法按钮，你希望它做什么？"
- "你愿意为这个功能付多少钱？"

#### 行为验证

- "你最近一次 [相关行为] 是什么时候？"
- "当时为什么选择这个方式？"
- "有没有尝试过其他方法？"

### 四、产品概念测试 (10 分钟，如适用)

- 展示原型/概念描述
- "这个能解决你的问题吗？"
- "哪里和你预期的不一样？"
- "你会向同事推荐吗？为什么？"

### 五、收尾 (5 分钟)

- "还有什么我没问到的想补充吗？"
- "身边有类似困扰的人吗？能介绍吗？"
- 感谢和后续跟进说明
```

#### 访谈问题设计原则

| 原则         | ✅ 好问题              | ❌ 坏问题            |
| ------------ | ---------------------- | -------------------- |
| **开放式**   | "能描述一下...吗？"    | "你喜欢这个功能吗？" |
| **具体场景** | "上次遇到是什么时候？" | "你通常会...吗？"    |
| **行为导向** | "你当时做了什么？"     | "你会...吗？"        |
| **避免引导** | "你对这个怎么看？"     | "这个很好用对吧？"   |

### 2. 用户画像构建模板

**Persona** - 基于真实数据的虚构用户代表

#### 画像要素

```markdown
## 用户画像 - [画像名称]

### 基本信息

| 维度     | 描述               |
| -------- | ------------------ |
| **姓名** | [虚构但真实的名字] |
| **年龄** | [年龄段]           |
| **职业** | [具体职位]         |
| **收入** | [收入范围]         |
| **地点** | [城市/区域]        |
| **家庭** | [家庭状况]         |

### 照片

[插入代表性照片]

### 个人标签

#标签 1 #标签 2 #标签 3

### 引言

> "[一句代表该用户典型想法的原话]"

### 目标与动机

| 类型         | 描述               |
| ------------ | ------------------ |
| **工作目标** | [工作中想达成什么] |
| **个人目标** | [个人生活目标]     |
| **产品期望** | [希望产品帮他什么] |

### 痛点与挫折

| 痛点     | 频率      | 影响程度 |
| -------- | --------- | -------- |
| [痛点 1] | 每天/每周 | 高/中/低 |
| [痛点 2] | 每天/每周 | 高/中/低 |

### 行为特征

- **使用场景**: [时间/地点/设备]
- **使用频率**: [每天/每周 X 次]
- **熟练程度**: [新手/中级/专家]
- **决策因素**: [价格/功能/品牌/推荐]

### 信息渠道

- **获取信息**: [公众号/知乎/朋友推荐]
- **信任来源**: [KOL/同事/官方]

### 典型一天

[描述该用户典型一天中产品可能介入的场景]

### 对我们的启示

- **产品设计**: [基于画像的设计建议]
- **功能优先级**: [P0/P1/P2 建议]
- **营销策略**: [如何触达这类用户]
```

#### 画像创建流程

1. **数据收集**: 访谈/问卷/行为数据
2. **模式识别**: 寻找共性特征和行为模式
3. **聚类分组**: 将用户分为 3-5 个典型群体
4. **丰富细节**: 为每个群体添加具体细节
5. **验证迭代**: 用新数据验证和更新画像

### 3. JTBD (Jobs-to-be-Done) 分析

**核心思想**: 用户不是买产品，而是"雇佣"产品完成某个任务

#### JTBD 框架

```
当 [情境] 时，
我想 [动机]，
以便 [期望结果]，
但是 [障碍]。
```

#### JTBD 分析模板

```markdown
## JTBD 分析 - [产品/功能]

### 核心 Job Statement

> 当 [用户情境] 时，我想 [核心动机]，以便 [期望结果]。

### Job 分解

#### 1. 功能性 Jobs

| Job          | 重要性   | 满意度   | 机会  |
| ------------ | -------- | -------- | ----- |
| [功能任务 1] | 高/中/低 | 高/中/低 | ✅/❌ |
| [功能任务 2] | 高/中/低 | 高/中/低 | ✅/❌ |

#### 2. 情感性 Jobs

| Job          | 描述             |
| ------------ | ---------------- |
| [情感需求 1] | [想感觉怎么样]   |
| [情感需求 2] | [想避免什么感觉] |

#### 3. 社交性 Jobs

| Job          | 描述                     |
| ------------ | ------------------------ |
| [社交需求 1] | [想在他人眼中是什么形象] |

### 用户故事地图
```

情境 → 动机 → 行动 → 结果 → 感受

```

### 竞争替代方案
| 替代方案 | 用户为什么选择 | 弱点 |
|---------|--------------|------|
| [方案 A] | | |
| [方案 B] | | |
| 什么都不做 | | |

### 创新机会
- **未满足的 Jobs**: [重要性高但满意度低]
- **过度满足的 Jobs**: [满意度高但重要性低]
- **差异化机会**: [竞争对手忽略的 Jobs]
```

#### JTBD 访谈问题

| 阶段     | 问题示例                                             |
| -------- | ---------------------------------------------------- |
| **情境** | "第一次考虑 [解决方案] 是什么时候？当时发生了什么？" |
| **动机** | "你当时最想解决什么问题？"                           |
| **选择** | "你考虑过哪些方案？为什么选这个？"                   |
| **结果** | "使用后有什么变化？达到预期了吗？"                   |

## 🔧 使用示例

### 示例 1: 生成用户访谈提纲

**输入**: "帮我生成企业 IM 产品的用户访谈提纲"

**输出**:

```markdown
## 用户访谈提纲 - 企业 IM 系统

### 一、暖场 (5 分钟)

- 自我介绍，说明访谈目的
- 询问受访者职位、团队规模、日常工作

### 二、背景了解 (10 分钟)

- "能描述一下你们团队目前的沟通方式吗？"
- "平时用什么工具进行工作沟通？"
- "最让你头疼的沟通问题是什么？"

### 三、核心问题 (25 分钟)

#### 痛点探索

- "能说说上次信息遗漏/延误的具体情况吗？"
- "当时造成了什么影响？"
- "你们是怎么解决的？"

#### 需求挖掘

- "理想的团队沟通工具应该具备什么？"
- "如果只能改进一点，你希望是什么？"
- "你愿意为更好的沟通体验付费吗？" ...
```

### 示例 2: 创建用户画像

**输入**: "帮我创建中小企业老板的用户画像"

**输出**:

```markdown
## 用户画像 - 张总 (中小企业主)

### 基本信息

| 维度         | 描述           |
| ------------ | -------------- |
| **年龄**     | 35-45 岁       |
| **职业**     | 创始人/CEO     |
| **公司规模** | 20-100 人      |
| **行业**     | 科技/服务/贸易 |
| **年收入**   | 50-200 万      |

### 引言

> "我最怕员工摸鱼，但又不能时刻盯着每个人。"

### 目标与动机

- **工作目标**: 提高团队效率，降低管理成本
- **痛点**: 信息不透明，进度难追踪
- **期望**: 一眼看清团队在做什么

### 启示

- **产品设计**: 强调管理视图和数据分析
- **营销**: 强调"降本增效"和"掌控感" ...
```

## 📋 输出规范

1. **基于真实数据**: 画像和 JTBD 需基于实际调研
2. **具体可操作**: 输出需能指导产品决策
3. **避免刻板印象**: 用数据而非偏见定义用户
4. **持续更新**: 随新数据迭代画像和洞察

---

## ⚠️ 注意事项

- ⛔ 避免凭空想象用户画像
- ⛔ 访谈问题避免引导性
- ⛔ JTBD 不是功能列表，是用户目标
- ✅ 至少访谈 5-8 个用户再形成结论
