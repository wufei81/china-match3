**版本**: 1.0 **创建日期**: 2026-03-26 **触发词**: "产品战略", "产品愿景", "路线图", "roadmap", "产品组合", "portfolio",

## 🎯 技能概述

为产品经理智能体提供产品战略能力，包括产品愿景画布、路线图规划、产品组合分析。帮助从"功能交付者"升级为"战略思考者"。

---

## 📊 核心功能

### 1. 产品愿景画布

**目的**: 清晰定义产品长期方向和核心价值主张

#### 愿景画布模板

```markdown
## 产品愿景画布 - [产品名称]

### 一、愿景陈述 (Vision Statement)

> 为 [目标用户] 解决 [核心问题] 提供 [解决方案] 不同于 [竞争者] 我们 [差异化价值]

**示例**:

> 为中小企业团队解决沟通效率低下、信息孤岛的问题提供一体化智能协作平台不同于传统 IM 工具我们深度融合 AI 助手，让协作更智能

### 二、目标用户

| 用户类型     | 描述   | 核心痛点 | 占比 |
| ------------ | ------ | -------- | ---- |
| **主要用户** | [描述] | [痛点]   | X%   |
| **次要用户** | [描述] | [痛点]   | Y%   |

### 三、核心价值主张

| 价值维度     | 描述                   | 重要性   |
| ------------ | ---------------------- | -------- |
| **功能价值** | [解决什么问题]         | 高/中/低 |
| **情感价值** | [让用户感觉如何]       | 高/中/低 |
| **社交价值** | [帮助用户建立什么形象] | 高/中/低 |

### 四、差异化优势

| 优势     | 描述   | 可持续性 |
| -------- | ------ | -------- |
| [优势 1] | [描述] | 高/中/低 |
| [优势 2] | [描述] | 高/中/低 |

### 五、成功指标

| 时间     | 成功标准   |
| -------- | ---------- |
| **1 年** | [量化目标] |
| **3 年** | [量化目标] |
| **5 年** | [愿景状态] |

### 六、战略边界 (不做什么)

| 不做事项 | 原因   |
| -------- | ------ |
| [不做 X] | [原因] |
| [不做 Y] | [原因] |
```

#### 愿景检验清单

| 问题           | 好愿景特征       |
| -------------- | ---------------- |
| 是否清晰？     | 一句话能说清楚   |
| 是否鼓舞人心？ | 能让团队兴奋     |
| 是否可实现？   | 不是空想，有路径 |
| 是否差异化？   | 与竞争者有区别   |
| 是否持久？     | 3-5 年不变       |

### 2. 路线图规划能力

**目的**: 将愿景转化为可执行的阶段性计划

#### 路线图框架

```
愿景 (3-5 年)
    ↓
战略主题 (1-2 年)
    ↓
季度目标 (OKR)
    ↓
迭代计划 (双周)
```

#### 路线图模板

```markdown
## 产品路线图 - [产品名称]

### 一、战略主题 (1-2 年)

| 主题       | 描述   | 成功指标 |
| ---------- | ------ | -------- |
| **主题 1** | [描述] | [指标]   |
| **主题 2** | [描述] | [指标]   |
| **主题 3** | [描述] | [指标]   |

### 二、季度路线图

| P0     | [功能 2]  | [主题]   | [成果]   | 规划中 |
| P1     | [功能 3]  | [主题]   | [成果]   | 研究中 |

#### Q2 2026

| 优先级 | 功能/项目 | 战略主题 | 预期成果 |

### 三、里程碑

| 时间   | 里程碑       | 成功标准 |
| ------ | ------------ | -------- |
| [日期] | [里程碑名称] | [标准]   |

### 四、资源分配

| 战略主题 | 研发资源 | 设计资源 | 预算    |
```

#### 路线图规划原则

| 原则         | 说明                         |
| ------------ | ---------------------------- |
| **远近结合** | 近期具体，远期方向性         |
| **主题驱动** | 按战略主题组织，不是功能列表 |
| **灵活调整** | 季度回顾，根据反馈调整       |
| **透明共享** | 全员可见，对齐期望           |
| **结果导向** | 定义成果，不是输出           |

### 3. 产品组合分析

**目的**: 管理多个产品/功能线，优化资源分配

#### 组合分析框架

```markdown
## 产品组合分析 - [产品线的名称]

### 一、组合概览

| 产品/功能 | 阶段   | 收入贡献 | 资源占比 | 战略重要性 |
| --------- | ------ | -------- | -------- | ---------- |
| [产品 A]  | 成长期 | X%       | X%       | 高/中/低   |
| [产品 B]  | 成熟期 | Y%       | Y%       | 高/中/低   |
| [产品 C]  | 引入期 | Z%       | Z%       | 高/中/低   |

### 二、BCG 矩阵分析
```

                    市场增长率
              高 ←───────────→ 低
            ┌─────────┬─────────┐
         高 │ 明星产品 │ 问题产品 │
    市       │ (Stars) │ (Question)│
    场   ───┼─────────┼─────────┤
    份      │ 现金牛  │ 瘦狗产品 │
    额      │ (Cash)  │ (Dogs)  │
            └─────────┴─────────┘

```

#### 各象限策略
| 象限 | 特征 | 策略 |
|------|------|------|
| **明星** | 高增长高份额 | 持续投入，保持领先 |
| **现金牛** | 低增长高份额 | 维持，产生现金流 |
| **问题** | 高增长低份额 | 选择性投资或放弃 |
| **瘦狗** | 低增长低份额 | 剥离或退出 |

### 三、资源分配建议
| 产品 | 当前资源 | 建议资源 | 调整理由 |
|------|---------|---------|---------|
| [产品 A] | X% | Y% | [理由] |
| [产品 B] | X% | Y% | [理由] |

### 四、产品生命周期管理
| 产品 | 当前阶段 | 下一阶段 | 预计时间 | 关键行动 |
|------|---------|---------|---------|---------|
| [产品] | 引入/成长/成熟/衰退 | [阶段] | [时间] | [行动] |

### 五、组合健康度
| 维度 | 评估 | 说明 |
|------|------|------|
| **收入多样性** | 好/中/差 | 是否过度依赖单一产品 |
| **增长引擎** | 好/中/差 | 是否有持续新产品 |
| **资源平衡** | 好/中/差 | 资源分配是否合理 |
| **战略对齐** | 好/中/差 | 是否支持公司战略 |
```

#### 产品组合决策框架

| 决策类型 | 考虑因素                       | 决策标准            |
| -------- | ------------------------------ | ------------------- |
| **投资** | 市场空间、竞争优势、ROI        | 预期 ROI > 阈值     |
| **维持** | 现金流、客户依赖、退出成本     | 现金流为正          |
| **剥离** | 战略不匹配、持续亏损、机会成本 | 资源可释放更高价值  |
| **孵化** | 战略重要性、学习价值、期权价值 | 战略价值 > 短期亏损 |

## 🔧 使用示例

### 示例 1: 创建产品愿景

**输入**: "帮企业 IM 产品写一个愿景"

**输出**:

```markdown
## 产品愿景 - 企业 IM 系统

### 愿景陈述

> 为中小企业团队解决沟通碎片化、协作效率低的问题提供 AI 驱动的一体化协作平台不同于传统 IM 工具我们让 AI 成为每个团队的协作助手

### 目标用户

| 用户类型     | 描述              | 核心痛点           |
| ------------ | ----------------- | ------------------ |
| **主要用户** | 20-200 人科技公司 | 信息分散、重复沟通 |
| **次要用户** | 传统企业转型中    | 数字化能力不足     |

### 核心价值

| 价值维度     | 描述                     |
| ------------ | ------------------------ |
| **功能价值** | 一体化平台，减少切换成本 |
| **情感价值** | 让工作更轻松，减少焦虑   |
| **社交价值** | 高效团队的专业形象       |

### 成功指标

| 时间     | 成功标准                   |
| -------- | -------------------------- |
| **1 年** | 10,000 活跃团队，NPS > 50  |
| **3 年** | 100,000 活跃团队，行业前三 |
| **5 年** | 成为中小企业协作首选平台   |

### 战略边界 (不做什么)

| 不做事项      | 原因                |
| ------------- | ------------------- |
| 不做 C 端社交 | 聚焦 B 端，资源有限 |
| 不做硬件      | 专注软件和服务      |
```

### 示例 2: 制定季度路线图

```markdown
## Q2 2026 产品路线图

| ------ | ------------ | -------- | ---------------- |
| P0     | AI 会议纪要  | 智能协作 | 会议效率提升 50% |
| P0     | SSO 单点登录 | 企业就绪 | 支持企业客户上线 |
| P1     | 数据看板     | 增长引擎 | 转化漏斗可视化   |
| P1     | 移动端优化   | 智能协作 | 移动端 DAU +30%  |

| 时间       | 里程碑           |
| ---------- | ---------------- |
| 4 月 15 日 | AI 会议纪要 Beta |

| 6 月 15 日 | Q2 目标复盘      |
```

## 📋 输出规范

1. **愿景清晰**: 一句话能说清楚产品方向
2. **路线图可执行**: 季度计划具体到功能和成果
3. **组合平衡**: 短期现金流 + 长期增长兼顾
4. **定期回顾**: 季度审视战略和路线图

---

## ⚠️ 注意事项

- ⛔ 愿景不是口号，要能指导日常决策
- ⛔ 路线图不是承诺，是方向性规划
- ⛔ 产品组合要定期审视，及时止损
- ✅ 战略要全员对齐，避免各自为战
