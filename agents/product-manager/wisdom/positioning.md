# April Dunford - Positioning Wisdom

**来源**: Lenny's Podcast  
**适用阶段**: Phase 1 - Problem Discovery / 竞品分析

---

## 💡 核心观点

> "Really great positioning feels so clear, so simple - of course that's what it is."

---

## 🎯 何时使用

- PRD Phase 1: 问题发现阶段
- 竞品分析时
- 定义产品差异化时

---

## ❓ 关键问题

在写 PRD 前，先回答这 5 个问题：

1. **竞争替代品有哪些？**
   - 包括"维持现状"（用户现在怎么解决这个问题）
   - 40% 的 B2B 交易输给"不做决定"

2. **我们提供什么独特价值？**
   - 不是功能列表，是用户能得到的好处

3. **谁最在乎这个价值？**
   - 找到最痛的用戶群体

4. **在什么场景下这个价值最重要？**
   - 场景决定价值

5. **如何证明这个价值？**
   - 客户案例、数据、对比

---

## 📋 PRD 应用模板

在 PRD 的"竞争格局"部分添加：

```markdown
### 竞争定位

| 维度 | 我们 | 竞品 A | 竞品 B | 维持现状 |
|------|------|--------|--------|----------|
| 目标用户 | | | | - |
| 核心价值 | | | | - |
| 关键场景 | | | | - |
| 差异化 | | | | - |
```

---

## 🚨 常见错误

- ❌ 把功能当定位（"我们有 XX 功能"）
- ❌ 试图满足所有人（"适合所有用户"）
- ❌ 忽略"维持现状"这个最大竞品
- ❌ 没有证明价值（只有声称，没有证据）

---

## ✅ 检查清单

写 PRD 时检查：

- [ ] 是否明确列出所有竞争替代方案（包括维持现状）
- [ ] 是否用一句话清晰描述独特价值
- [ ] 是否明确目标用户群体
- [ ] 是否有证据证明价值（案例/数据）
- [ ] 是否避免功能列表式定位
