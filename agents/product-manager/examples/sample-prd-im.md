## 1. 问题陈述

### 1.1 核心问题
企业团队需要轻量级即时通讯解决方案，用于内部快速沟通与协作。

**用户原话**（推断）:
> "微信群太乱，钉钉太重，想要个简单的内部沟通工具"

### 1.2 Why Now（紧迫性）

| 因素 | 说明 | 置信度 |
|------|------|--------|
| 团队扩张 | ⚠️ 假设团队达 20+ 人，现有工具效率下降 | 65% |
| 远程办公 | 混合办公成为常态，沟通需求增加 | 80% |
| 数据安全 | 企业对数据隐私关注度提升 | 75% |

**如果不解决会怎样**:
- 沟通效率持续下降
- 信息分散在多个平台
- 敏感数据可能泄露到外部平台

### 1.3 现有解决方案

| 方案类型 | 具体方案 | 缺点/局限 |
|----------|----------|-----------|
| 竞品 | 企业微信/钉钉 | 功能过重，学习成本高 |
| 竞品 | 微信群 | 信息混杂，无组织结构 |
| 竞品 | Slack/Teams | 成本高，国内访问不稳定 |
| 维持现状 | 邮件 + 微信 | 效率低，信息分散 |

## 5. 功能需求详情

### 5.1 功能：即时消息

**功能 ID**: FEAT-001

**优先级**: P0

**功能描述**:
支持用户之间发送文本、表情、图片消息，实时送达。

**用户流程**:
```
用户 A 输入消息 → 点击发送 → 消息显示（发送中）→ 
服务器确认 → 消息显示（已送达）→ 用户 B 收到推送
```

**验收标准**（Gherkin）:
```gherkin
Feature: 即时消息

Scenario: 发送文本消息
  Given 用户 A 和 B 已建立联系
  When 用户 A 发送文本消息
  Then 消息在 200ms 内显示在 B 的对话框
  And 消息状态显示为"已送达"

Scenario: 网络异常时发送消息
  Given 网络不稳定
  When 用户 A 发送消息
  Then 消息显示"发送失败"
  And 显示"重试"按钮

Scenario: 发送图片消息
  Given 用户选择图片
  When 用户发送图片
  Then 图片压缩后上传
  And 在对方对话框显示缩略图
```

**UI/UX 要求**:
- [ ] 消息气泡样式（左收右发）
- [ ] 发送中/已送达/已读状态图标
- [ ] 图片支持点击放大
- [ ] 表情面板

**数据要求**:
- [ ] 消息持久化存储（6 个月）
- [ ] 图片存储（CDN 加速）
- [ ] 消息状态实时更新

## 6. 数据结构草稿

### 6.1 消息表 (messages)

```sql
CREATE TABLE messages (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  sender_id BIGINT NOT NULL,
  receiver_id BIGINT,
  group_id BIGINT,
  content TEXT NOT NULL,
  message_type ENUM('text', 'image', 'emoji') DEFAULT 'text',
  status ENUM('sending', 'sent', 'delivered', 'read', 'failed') DEFAULT 'sending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_sender (sender_id),
  INDEX idx_receiver (receiver_id),
  INDEX idx_group (group_id),
  INDEX idx_created (created_at)
);
```

### 6.2 群组表 (groups)

```sql
CREATE TABLE groups (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  owner_id BIGINT NOT NULL,
  description TEXT,
  max_members INT DEFAULT 50,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_owner (owner_id)
);
```

### 6.3 群组成员表 (group_members)

```sql
CREATE TABLE group_members (
  group_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  role ENUM('owner', 'admin', 'member') DEFAULT 'member',
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (group_id, user_id),
  FOREIGN KEY (group_id) REFERENCES groups(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 7. API 契约草稿

### 7.1 发送消息

```yaml
POST /api/v1/messages
Content-Type: application/json

Request:
{
  "receiver_id": 123,          // 可选，私聊时必填
  "group_id": 456,             // 可选，群聊时必填
  "content": "Hello",          // 必填
  "message_type": "text"       // 必填：text|image|emoji
}

Response (201 Created):
{
  "message_id": 789,
  "status": "sent",
  "created_at": "2026-03-26T10:00:00Z"
}

Response (400 Bad Request):
{
  "error": "invalid_request",
  "message": "receiver_id or group_id is required"
}
```

### 7.2 获取消息历史

```yaml
GET /api/v1/messages?receiver_id=123&limit=50&cursor=xxx

Response (200 OK):
{
  "messages": [
    {
      "id": 789,
      "sender_id": 100,
      "content": "Hello",
      "message_type": "text",
      "status": "read",
      "created_at": "2026-03-26T10:00:00Z"
    }
  ],
  "next_cursor": "yyy",
  "has_more": true
}
```
