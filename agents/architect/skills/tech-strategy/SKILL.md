## 🎯 技能概述

为架构师智能体提供技术战略规划能力，包括技术雷达、技术路线图、成本优化 (FinOps)、技术债务管理。帮助从"项目架构师"升级为"组织技术战略家"。

---

## 📊 核心功能

### 1. 技术雷达 (Technology Radar)

**目的**: 追踪和评估新兴技术，指导组织技术选型

#### 技术雷达结构

| 环 | 描述 | 行动建议 |
|----|------|---------|
| **采纳 (Adopt)** | 成熟可靠，大规模使用 | 积极采用，建立标准 |
| **试验 (Trial)** | 有前景，小范围试点 | 在项目中尝试 |
| **评估 (Assess)** | 值得关注，持续观察 | 学习研究，评估价值 |
| **保留 (Hold)** | 不推荐，逐步淘汰 | 避免使用，考虑迁移 |

#### 技术雷达模板

```markdown
## 技术雷达 - [组织名称] [年份.QX]

### 方法论
- **采纳**: 已在生产环境大规模使用
- **试验**: 在 1-2 个项目试点
- **评估**: 持续关注，适合特定场景
- **保留**: 不推荐新项目使用

### 技术分类
- **语言 & 框架**
- **平台 & 基础设施**
- **工具 & 实践**
- **数据 & AI**

### 本期变化
| 技术 | 变化 | 原因 |
|------|------|------|
| [技术 A] | 评估 → 试验 | 试点项目成功 |
| [技术 B] | 试验 → 采纳 | 大规模验证 |
| [技术 C] | 采纳 → 保留 | 维护成本高 |
| [技术 D] | 新增 - 评估 | 新兴技术 |

### 详细评估
#### 语言 & 框架
| 技术 | 环 | 描述 | 适用场景 |
|------|-----|------|---------|
| TypeScript | 采纳 | 前端标准语言 | 所有前端项目 |
| Go | 采纳 | 后端主力语言 | 微服务、API |
| Rust | 试验 | 高性能场景 | 系统编程 |
| [新技术] | 评估 | 持续关注 | - |

#### 平台 & 基础设施
| 技术 | 环 | 描述 |
|------|-----|------|
| Kubernetes | 采纳 | 容器编排标准 |
| Docker | 采纳 | 容器化标准 |
| Serverless | 试验 | 特定场景使用 |
```

#### 技术评估框架

| 维度 | 评估项 | 权重 |
|------|-------|------|
| **成熟度** | 社区活跃度、生产案例 | 25% |
| **技术优势** | 性能、易用性、特性 | 25% |
| **生态** | 文档、工具、人才 | 20% |
| **成本** | 学习成本、运维成本 | 15% |
| **风险** | 供应商锁定、安全性 | 15% |

### 3. 成本优化 (FinOps)

**目的**: 优化云资源成本，提高资源使用效率

#### 成本优化框架

```markdown
## 成本优化方案 - [组织名称]

### 成本现状分析
#### 成本结构
| 类别 | 月成本 | 占比 |
|------|-------|------|
| 计算 (EC2/ECS) | ¥XX 万 | X% |
| 存储 (S3/EBS) | ¥XX 万 | X% |
| 网络 (带宽/CDN) | ¥XX 万 | X% |
| 数据库 (RDS) | ¥XX 万 | X% |
| 其他 | ¥XX 万 | X% |
| **总计** | ¥XX 万 | 100% |

#### 成本趋势
| 月份 | 成本 | 环比 |
|------|------|------|
| 1 月 | ¥XX 万 | - |
| 2 月 | ¥XX 万 | +X% |
| 3 月 | ¥XX 万 | +X% |

### 优化策略
#### 快速见效 (0-1 月)
| 措施 | 预计节省 | 实施难度 |
|------|---------|---------|
| 闲置资源清理 | ¥X 万/月 | 低 |
| 预留实例购买 | ¥X 万/月 | 低 |
| 存储分层 | ¥X 万/月 | 低 |

#### 中期优化 (1-3 月)
| 措施 | 预计节省 | 难度 |
|------|---------|------|
| 自动伸缩优化 | ¥X 万/月 | 中 |
| 容器化改造 | ¥X 万/月 | 中 |
| 数据库优化 | ¥X 万/月 | 中 |

#### 长期优化 (3-6 月)
| 措施 | 预计节省 | 难度 |
|------|---------|------|
| 架构重构 | ¥X 万/月 | 高 |
| 混合云部署 | ¥X 万/月 | 高 |
| 自研替代 | ¥X 万/月 | 高 |

### 成本分摊
| 部门 | 分摊比例 | 月成本 |
|------|---------|-------|
| 产品 A | X% | ¥X 万 |
| 产品 B | X% | ¥X 万 |
| 平台部 | X% | ¥X 万 |

### 目标与追踪
- **目标**: 6 个月内成本降低 30%
- **月度追踪**: [看板链接]
- **责任人**: [姓名]
```

#### 成本优化检查清单

| 领域 | 检查项 | 节省潜力 |
|------|-------|---------|
| **计算** | 实例类型优化、预留实例、自动伸缩 | 20-40% |
| **存储** | 存储分层、生命周期管理、压缩 | 30-50% |
| **网络** | CDN 优化、流量整形、对等连接 | 10-30% |
| **数据库** | 实例优化、读写分离、缓存 | 20-40% |
| **监控** | 日志保留策略、指标采样 | 10-20% |

### 4. 技术影响力建设

**目的**: 提升组织技术影响力，吸引和培养人才

#### 技术影响力框架

```markdown
## 技术影响力建设 - [组织名称]

### 内部建设
#### 技术分享
- **频率**: 每周/双周
- **形式**: 内部分享、技术沙龙
- **主题**: 技术深度、项目复盘、行业趋势

#### 技术培训
| 类型 | 对象 | 内容 |
|------|------|------|
| 新人培训 | 入职 0-3 月 | 技术栈、规范、流程 |
| 技能培训 | 全员 | 专项技术深入 |
| 领导力培训 | Tech Lead | 团队管理、架构决策 |

#### 代码文化
- **Code Review**: 每 PR 必 review
- **技术文档**: ADR、RFC 文化
- **开源项目**: 内部开源、外部贡献

### 外部建设
#### 技术品牌
- **技术博客**: 月度更新
- **技术大会**: 年度演讲
- **开源贡献**: GitHub 项目

#### 行业参与
| 活动 | 频率 | 目标 |
|------|------|------|
| 技术大会 | 季度 | 提升知名度 |
| 技术沙龙 | 月度 | 社区建设 |
| 高校合作 | 年度 | 人才储备 |

### 人才发展
#### 职级体系
| 级别 | 能力要求 | 晋升标准 |
|------|---------|---------|
| 初级 | 独立完成模块 | 代码质量、学习速度 |
| 中级 | 独立负责服务 | 系统设计、问题解决 |
| 高级 | 负责系统架构 | 技术决策、影响力 |
| 专家 | 负责技术方向 | 战略规划、行业影响 |

#### 导师制度
- **配对**: 资深员工 + 新员工
- **周期**: 3-6 个月
- **目标**: 快速融入、技能提升
```

## 🔧 使用示例

### 示例 1: 制定技术雷达

**输入**: "帮我们制定 2026 年技术雷达"

**输出**:
```markdown
## 技术雷达 2026.Q1

### 采纳 (Adopt)
- **语言**: TypeScript, Go, Python
- **框架**: React, Spring Boot, FastAPI
- **平台**: Kubernetes, Docker, AWS
- **数据库**: PostgreSQL, Redis, MongoDB

### 试验 (Trial)
- **语言**: Rust (系统编程场景)
- **框架**: Next.js 14 (新项目)
- **AI**: LLM 应用开发

### 评估 (Assess)
- **技术**: WebAssembly, Edge Computing
- **数据库**: ClickHouse (分析场景)

### 保留 (Hold)
- **框架**: Angular (新项目不推荐)
- **数据库**: MySQL (迁移到 PostgreSQL)
```

### 示例 2: 成本优化方案

**输入**: "云成本太高，帮忙制定优化方案"

**输出**:
```markdown
## 成本优化方案

### 现状
- **月成本**: ¥50 万
- **主要构成**: 计算 40%, 数据库 25%, 存储 15%, 网络 20%

### 优化措施
#### 快速见效 (本月)
1. 清理闲置资源 → 节省¥5 万/月
2. 购买预留实例 → 节省¥8 万/月
3. 存储分层 → 节省¥3 万/月

#### 中期优化 (3 个月)
1. 容器化改造 → 节省¥10 万/月
2. 自动伸缩优化 → 节省¥5 万/月

### 目标
- **3 个月**: 成本降低 30% (¥50 万→¥35 万)
- **6 个月**: 成本降低 40% (¥50 万→¥30 万)
```

## 📋 输出规范

1. **数据驱动**: 技术决策基于实际数据
2. **可执行**: 路线图有明确责任人和时间
3. **可衡量**: 成本优化有明确目标
4. **定期回顾**: 季度更新技术雷达

---

## ⚠️ 注意事项

- ⛔ 技术雷达不是技术堆砌，要结合实际
- ⛔ 成本优化不是越低越好，平衡性能和成本
- ⛔ 技术路线不是僵化的，要灵活调整
- ✅ 技术战略要获得高层支持和资源投入
