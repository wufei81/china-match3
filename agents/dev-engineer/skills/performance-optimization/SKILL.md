## 📊 核心功能

### 1. 性能分析方法论

**目的**: 科学定位性能瓶颈，避免盲目优化

#### 性能优化流程

```
1. 建立基线
   - 定义性能指标
   - 测量当前性能
   - 设定优化目标

2. 定位瓶颈
   - Profiling 分析
   - 识别热点代码
   - 确定优化优先级

3. 实施优化
   - 针对性优化
   - 小步验证
   - 回归测试

4. 验证效果
   - 对比基线
   - 确认达到目标
   - 监控线上表现
```

#### 性能指标定义

| 指标 | 定义 | 目标 |
|------|------|------|
| **响应时间** | 请求到响应的时间 | P95 < 200ms |
| **吞吐量** | 单位时间处理请求数 | QPS > 1000 |
| **错误率** | 失败请求比例 | < 0.1% |
| **资源利用率** | CPU/内存使用率 | CPU < 70%, Mem < 80% |
| **并发能力** | 同时处理请求数 | 并发 > 100 |

#### Profiling 工具

| 语言 | 工具 | 用途 |
|------|------|------|
| **Node.js** | clinic.js, 0x, --inspect | CPU/内存分析 |
| **Python** | cProfile, py-spy, memory_profiler | 性能分析 |
| **Java** | JProfiler, VisualVM, async-profiler | JVM 分析 |
| **Go** | pprof | CPU/内存分析 |
| **前端** | Chrome DevTools, Lighthouse | 页面性能 |

### 2. 后端性能优化

**目的**: 优化服务端性能，提升吞吐和响应

#### 数据库优化

```markdown
## 数据库优化清单

### 查询优化
- [ ] 添加必要索引
- [ ] 避免 SELECT *
- [ ] 避免 N+1 查询
- [ ] 使用 EXPLAIN 分析执行计划
- [ ] 优化 JOIN 顺序

### 架构优化
- [ ] 读写分离
- [ ] 分库分表 (大数据量)
- [ ] 连接池配置优化
- [ ] 慢查询监控

### 缓存策略
- [ ] 热点数据缓存
- [ ] 缓存穿透防护 (布隆过滤器)
- [ ] 缓存雪崩防护 (随机 TTL)
- [ ] 缓存击穿防护 (互斥锁)
```

#### 代码层优化

| 优化点 | 说明 | 示例 |
|------|------|------|
| **算法优化** | 降低时间复杂度 | O(n²) → O(n log n) |
| **批量操作** | 减少网络往返 | 批量插入 vs 单条插入 |
| **异步处理** | 非阻塞 IO | async/await, 消息队列 |
| **连接复用** | 减少连接开销 | 连接池，HTTP Keep-Alive |
| **数据压缩** | 减少传输量 | Gzip, Protobuf |

#### 优化案例

```javascript
// ❌ 优化前：N+1 查询
for (const user of users) {
  const orders = await db.orders.findByUserId(user.id);
  user.orders = orders;
}

// ✅ 优化后：批量查询
const userIds = users.map(u => u.id);
const orders = await db.orders.findAll({ where: { userId: userIds } });
// 内存关联
users.forEach(user => {
  user.orders = orders.filter(o => o.userId === user.id);
});
```

### 3. 前端性能优化

**目的**: 优化页面加载和交互体验

#### 核心 Web 指标

| 指标 | 描述 | 目标 |
|------|------|------|
| **LCP** (最大内容绘制) | 主要内容加载时间 | < 2.5s |
| **FID** (首次输入延迟) | 首次交互响应时间 | < 100ms |
| **CLS** (累计布局偏移) | 视觉稳定性 | < 0.1 |
| **FCP** (首次内容绘制) | 首次渲染时间 | < 1.8s |
| **TTI** (可交互时间) | 完全可交互时间 | < 3.8s |

#### 优化清单

```markdown
## 前端性能优化清单

### 资源优化
- [ ] 代码压缩 (Terser, cssnano)
- [ ] 图片优化 (WebP, 懒加载)
- [ ] 字体优化 (WOFF2, 子集化)
- [ ] 代码分割 (按需加载)
- [ ] Tree Shaking (移除死代码)

### 加载优化
- [ ] CDN 加速
- [ ] HTTP/2 或 HTTP/3
- [ ] 预加载关键资源
- [ ] 服务端渲染 (SSR)
- [ ] 静态资源缓存

### 渲染优化
- [ ] 避免强制同步布局
- [ ] 使用 CSS transforms
- [ ] 虚拟列表 (大数据)
- [ ] 防抖节流
- [ ] Web Worker (重计算)

### 包体积优化
- [ ] 分析包体积 (webpack-bundle-analyzer)
- [ ] 按需引入组件库
- [ ] 动态导入大模块
- [ ] 移除未使用依赖
```

#### 优化案例

```javascript
// ❌ 优化前：重复渲染
function UserList({ users }) {
  return (
    <div>
      {users.map(user => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}

// ✅ 优化后：React.memo + useMemo
const UserList = React.memo(({ users }) => {
  return (
    <div>
      {users.map(user => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
});
```

## ⚠️ 注意事项

- ⛔ 先 profiling 再优化 (不要猜测)
- ⛔ 避免过早优化 (Knuth 原则)
- ⛔ 优化后回归测试
- ✅ 性能是持续过程，不是一次性任务
