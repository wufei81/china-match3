**版本**: 1.0  
**创建日期**: 2026-03-26

---

## 🎯 技能概述

为 UI/UX 设计师智能体提供设计验证能力，包括可用性测试、A/B 测试、设计评审、可访问性审计。帮助从"主观设计"升级为"数据验证设计"。

---

## 📊 核心功能

### 1. 可用性测试

**目的**: 验证设计的可用性和用户体验

#### 测试流程

```
1. 定义目标 → 2. 招募用户 → 3. 执行测试 → 4. 分析结果 → 5. 优化设计
```

---

### 2. A/B 测试设计

**目的**: 数据驱动设计决策

#### A/B 测试模板

```markdown
## A/B 测试方案

### 假设
- **假设**: [如果做 X，那么 Y 会提升 Z%]

### 变体
| 变体 | 改动 | 预期影响 |
|------|------|---------|
| A (对照) | 当前设计 | - |
| B (实验) | [改动描述] | [预期提升] |

### 指标
- **核心指标**: [指标名称]
- **护栏指标**: [指标名称]

### 成功标准
- 统计显著：p < 0.05
- 实际显著：提升>X%
```

---

### 3. 可访问性审计

**目的**: 确保设计符合 WCAG 标准

#### 审计清单

| 检查项 | 标准 | 验证方法 |
|-------|------|---------|
| 颜色对比度 | ≥4.5:1 | 对比度工具 |
| 键盘导航 | 全部可访问 | 手动测试 |
| 屏幕阅读器 | 正确朗读 | NVDA/JAWS |
| 焦点可见 | 清晰可见 | 视觉检查 |
