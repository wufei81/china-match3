**版本**: 1.0  
**创建日期**: 2026-03-26  
**触发词**: "用户研究", "用户体验", "用户旅程", "user research", "UX", "user journey"

---

## 🎯 技能概述

为 UI/UX 设计师智能体提供用户研究能力，包括用户画像、用户旅程图、可用性测试、体验地图。帮助从"视觉设计"升级为"用户体验设计"。

---

## 📊 核心功能

### 1. 用户画像 (Persona)

**目的**: 基于真实用户数据创建代表性用户模型

#### 画像模板

```markdown
## 用户画像 - [名称]

### 基本信息
- **姓名**: [虚构名字]
- **年龄**: X-X 岁
- **职业**: [具体职位]
- **收入**: [范围]

### 引言
> "[代表性语录]"

### 目标与痛点
| 目标 | 痛点 |
|------|------|
| [工作目标] | [当前困难] |
| [个人目标] | [阻碍因素] |

### 行为特征
- **使用场景**: [时间/地点/设备]
- **使用频率**: [每天/每周]
- **熟练程度**: [新手/中级/专家]

### 设计启示
- [基于画像的设计建议]
```

---

### 2. 用户旅程图 (User Journey Map)

**目的**: 可视化用户体验全流程，识别痛点

#### 旅程图模板

```markdown
## 用户旅程 - [场景名称]

### 阶段划分
| 阶段 | 行为 | 想法 | 情绪 | 痛点 | 机会 |
|------|------|------|------|------|------|
| 认知 | [行为] | [想法] | 😐 | [痛点] | [机会] |
| 考虑 | [行为] | [想法] | 😊 | [痛点] | [机会] |
| 使用 | [行为] | [想法] | 😄 | [痛点] | [机会] |
| 留存 | [行为] | [想法] | 😐 | [痛点] | [机会] |

### 情绪曲线
[情绪随阶段变化曲线]

### 关键洞察
- [洞察 1]
- [洞察 2]

### 设计机会
- [机会 1]
- [机会 2]
```

---

### 3. 可用性测试

**目的**: 验证设计方案的有效性

#### 测试模板

```markdown
## 可用性测试计划

### 测试目标
- [目标 1]
- [目标 2]

### 参与者
- **人数**: 5-8 人
- **画像**: [目标用户特征]

### 任务列表
| 任务 | 成功标准 | 时间限制 |
|------|---------|---------|
| [任务 1] | [标准] | X 分钟 |
| [任务 2] | [标准] | X 分钟 |

### 指标
| 指标 | 目标 |
|------|------|
| 任务完成率 | ≥90% |
| 平均完成时间 | <X 分钟 |
| 错误次数 | <X 次 |
| SUS 评分 | ≥70 |

### 发现与建议
| 问题 | 严重性 | 建议 |
|------|-------|------|
| [问题] | 高/中/低 | [建议] |
```

## 🔧 使用示例

### 示例 1: 创建用户旅程

**输入**: "创建电商购物用户旅程"

**输出**:
```markdown
## 用户旅程 - 电商购物

### 阶段
1. **浏览**: 首页/搜索 → 发现商品
2. **考虑**: 查看详情/对比 → 加入购物车
3. **购买**: 结算/支付 → 完成订单
4. **售后**: 物流/评价 → 复购

### 痛点
- 搜索不精准
- 对比困难
- 结算流程长

### 机会
- 智能推荐
- 对比工具
- 一键下单
```
