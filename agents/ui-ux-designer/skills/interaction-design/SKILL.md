## 🎯 技能概述

为 UI/UX 设计师智能体提供交互设计能力，包括交互模式、状态设计、动效规范、微交互。帮助从"静态设计"升级为"动态交互设计"。

---

## 📊 核心功能

### 1. 交互模式库

**目的**: 标准化交互行为，提高一致性

#### 常见交互模式

| 模式 | 用途 | 示例 |
|------|------|------|
| **点击反馈** | 确认操作 | 按钮按下效果 |
| **加载状态** | 等待反馈 | 骨架屏、加载动画 |
| **错误处理** | 错误提示 | Toast、弹窗 |
| **空状态** | 无数据提示 | 引导操作 |
| **成功反馈** | 完成确认 | 对勾动画 |

---

### 2. 状态设计规范

**目的**: 定义组件所有状态，避免遗漏

#### 状态清单

```markdown
## 状态规范 - [组件名称]

### 必定义状态
| 状态 | 触发条件 | 视觉表现 | 交互行为 |
|------|---------|---------|---------|
| **Default** | 默认 | [样式] | [行为] |
| **Hover** | 鼠标悬停 | [样式] | [行为] |
| **Focus** | 键盘聚焦 | [样式] | [行为] |
| **Active** | 点击/按下 | [样式] | [行为] |
| **Disabled** | 禁用 | [样式] | 无 |
| **Loading** | 加载中 | [样式] | 禁用 |
| **Error** | 错误 | [样式] | [行为] |
| **Empty** | 无数据 | [样式] | [引导] |
```

---

### 3. 动效设计规范

**目的**: 标准化动画行为，提升体验

#### 动效原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **目的性** | 动效要有明确目的 | 加载反馈 |
| **自然性** | 符合物理规律 | 缓动曲线 |
| **适度性** | 不过度使用 | 时长<300ms |
| **一致性** | 全系统统一 | 统一缓动 |

#### 动效规范

```json
{
  "duration": {
    "fast": "150ms",
    "normal": "250ms",
    "slow": "400ms"
  },
  "easing": {
    "ease-in-out": "cubic-bezier(0.4, 0, 0.2, 1)",
    "ease-out": "cubic-bezier(0, 0, 0.2, 1)"
  },
  "animation": {
    "fade-in": "opacity 0.25s ease-in-out",
    "slide-up": "transform 0.25s ease-out"
  }
}
```
