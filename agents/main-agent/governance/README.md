## 🚀 快速开始

### 新项目启动

```bash
# 1. 复制治理文档
cp -r /path/to/governance /your/project/.governance/

# 2. 执行项目启动清单
cat .governance/01-project-checklist.md

# 3. 配置 Pre-commit
pre-commit install

# 4. 分配智能体职责
cat .governance/06-agent-responsibilities.md
```

### 代码提交前

```bash
# 1. 运行 Pre-commit 检查
pre-commit run --all-files

# 2. 对照审查清单自查
cat .governance/02-code-review-checklist.md

# 3. 提交代码
git commit -m "feat: add feature"
```

### API 开发时

```bash
# 1. 阅读 API 设计规范
cat .governance/04-api-design-standard.md

# 2. 按模板创建 API
# 3. 注册到 main.py
# 4. 验证 API 文档
curl http://localhost:8000/docs
```

## 🎯 核心原则

### 1. 自动化优先

> **能自动检查的，绝不人工**

- Pre-commit 自动检查
- CI/CD 自动测试
- 质量门禁自动拦截

### 2. 文档化驱动

> **没有文档的，视为不存在**

- 架构设计文档
- API 文档
- 代码审查记录

### 3. 审查常态化

> **没有审查的代码，不能合并**

- 100% 代码审查覆盖
- 双人审查机制
- 审查意见可追溯

### 4. 持续改进

> **今天的最佳实践，明天的起点**

- 定期回顾
- 知识沉淀
- 机制优化
