# 代码审查清单 (Code Review Checklist)

**适用范围:** 所有代码提交 **执行时机:** 每次 Pull Request / 代码合并前 **审查人:** tech-lead + 相关智能体

---

## 🔍 审查流程

```
代码提交
    ↓
自动检查 (CI)
    ↓
智能体审查 (分配审查人)
    ↓
问题修复 (如有)
    ↓
最终批准
    ↓
合并到主分支
```

---

## ✅ 自动检查项 (CI 执行)

### 1. 语法检查

- [ ] Python: flake8 通过
- [ ] JavaScript: ESLint 通过
- [ ] 无语法错误
- [ ] 无未使用导入

### 2. 代码格式化

- [ ] Black / Prettier 格式化
- [ ] 缩进一致 (4 空格/2 空格)
- [ ] 行宽限制 (120 字符)
- [ ] 文件末尾空行

### 3. 导入检查

- [ ] 导入语句排序正确
- [ ] 无循环导入
- [ ] 标准库优先
- [ ] 第三方库次之
- [ ] 本地模块最后

### 4. 安全扫描

- [ ] 无硬编码密码
- [ ] 无敏感信息
- [ ] 无 SQL 注入风险
- [ ] 依赖安全扫描通过

---

## ✅ 代码质量审查

### 5. 代码结构

- [ ] 函数长度 < 50 行
- [ ] 类长度 < 300 行
- [ ] 文件长度 < 500 行
- [ ] 圈复杂度 < 10

### 6. 命名规范

- [ ] 变量名：snake_case (Python) / camelCase (JS)
- [ ] 类名：PascalCase
- [ ] 常量：UPPER_CASE
- [ ] 私有成员：\_prefix

### 7. 注释文档

- [ ] 公共函数有 docstring
- [ ] 复杂逻辑有注释
- [ ] TODO 标记清晰
- [ ] API 文档完整

### 8. 错误处理

- [ ] 异常捕获具体
- [ ] 错误信息清晰
- [ ] 日志记录适当
- [ ] 资源正确释放

## 📊 审查评分标准

| 评分     | 标准           | 操作       |
| -------- | -------------- | ---------- |
| **5 分** | 优秀，无需修改 | 直接批准   |
| **4 分** | 良好，小问题   | 修复后批准 |
| **3 分** | 合格，需改进   | 修复后复审 |
| **2 分** | 不合格，大问题 | 重新设计   |
| **1 分** | 拒绝，严重问题 | 关闭 PR    |

**合并标准:** 平均分 ≥ 4 分 且 无 1 分项

## 🎯 智能体审查职责

| 智能体              | 审查重点            | 工具     |
| ------------------- | ------------------- | -------- |
| **tech-lead**       | 架构设计 + 代码质量 | 总体审查 |
| **architect**       | 设计模式 + 模块解耦 | 架构审查 |
| **dev-engineer**    | 代码规范 + 实现细节 | 代码审查 |
| **qa-engineer**     | 测试覆盖 + 边界条件 | 测试审查 |
| **devops-engineer** | 部署配置 + 监控     | 运维审查 |
| **security**        | 安全漏洞 + 敏感信息 | 安全扫描 |

## 🚀 自动化审查工具

### Python

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.4.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml

  - repo: https://github.com/psf/black
    rev: 23.1.0
    hooks:
      - id: black

  - repo: https://github.com/pycqa/flake8
    rev: 6.0.0
    hooks:
      - id: flake8

  - repo: https://github.com/pycqa/isort
    rev: 5.12.0
    hooks:
      - id: isort
```

### JavaScript

```yaml
# .eslintrc.js
module.exports = {
  env: {
    browser: true,

    'no-unused-vars': 'error',
    'no-console': 'warn',
  },
}
```

## 📈 审查指标追踪

| 指标         | 目标值   | 测量方式           |
| ------------ | -------- | ------------------ |
| 审查覆盖率   | 100%     | PR 数量/审查数量   |
| 平均审查时间 | < 4 小时 | 提交到批准时间     |
| 一次通过率   | > 60%    | 无需修改的 PR 比例 |
| 缺陷发现率   | > 90%    | 审查发现的问题数   |
| 代码质量分   | > 8/10   | 平均审查评分       |
