## 🎯 技能概述

为 Tech Leader 智能体提供代码质量管理能力，包括质量体系设计、代码审查流程、质量标准、自动化工具链。帮助从"代码审查"升级为"质量体系建设"。

---

## 📊 核心功能

### 1. 质量体系设计

**目的**: 建立全面的代码质量保障体系

#### 质量层次

```
┌─────────────────────────────────┐
│       业务质量 (E2E 测试)        │
├─────────────────────────────────┤
│       集成质量 (集成测试)        │
├─────────────────────────────────┤
│       单元质量 (单元测试)        │
├─────────────────────────────────┤
│       代码质量 (Lint/审查)       │
└─────────────────────────────────┘
```

#### 质量门禁

| 门禁 | 标准 | 工具 |
|------|------|------|
| **代码规范** | 0 Error, <10 Warning | ESLint/Prettier |
| **单元测试** | 覆盖率≥80% | Jest/Pytest |
| **集成测试** | 100% Pass | CI/CD |
| **E2E 测试** | P0 100% Pass | Playwright |
| **安全扫描** | 0 Critical, 0 High | Snyk/SonarQube |

---

### 2. 代码审查流程

**目的**: 系统化代码审查，确保质量

#### 审查清单

```markdown
## 代码审查清单

### 功能正确性
- [ ] 实现符合需求
- [ ] 边界条件处理
- [ ] 错误处理完整

### 代码质量
- [ ] 函数职责单一
- [ ] 命名清晰
- [ ] 无重复代码

### 安全性
- [ ] 输入验证
- [ ] 无注入风险
- [ ] 敏感数据保护

### 性能
- [ ] 无 N+1 查询
- [ ] 资源正确释放
- [ ] 缓存使用合理
```

---

### 3. 质量度量

**目的**: 量化代码质量，持续改进

#### 质量指标

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| **代码覆盖率** | ≥80% | 测试工具 |
| **代码异味数** | 0 | 静态分析 |
| **技术债务比** | ≤5% | SonarQube |
| **审查覆盖率** | 100% | Git 统计 |
| **Bug 密度** | <5/KLOC | Bug 追踪 |
