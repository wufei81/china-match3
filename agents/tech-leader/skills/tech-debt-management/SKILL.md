为 Tech Leader 智能体提供技术债务管理能力，包括债务识别、量化、偿还计划、预防机制。帮助从"被动修复"升级为"主动管理"。

---

## 📊 核心功能

### 1. 债务识别

**目的**: 系统化识别技术债务

#### 债务类别

| 类别 | 示例 |
|------|------|
| **代码债务** | 重复代码、复杂度高 |
| **设计债务** | 耦合严重、边界模糊 |
| **测试债务** | 覆盖率低、无自动化 |
| **文档债务** | 文档缺失、过时 |
| **基础设施债务** | 手动部署、无监控 |

---

### 2. 债务量化

**目的**: 量化债务影响，确定优先级

#### 量化模型

```
技术债务利息 = 额外维护时间 × 人力成本

技术债务本金 = 重构所需时间 × 人力成本

ROI = (利息累积 - 本金) / 本金 × 100%
```

#### 债务登记册

| ID | 债务 | 利息/月 | 本金 | 优先级 |
|----|------|--------|------|-------|
| TD-001 | 重复代码 | 10h | 20h | P1 |
| TD-002 | 无测试 | 15h | 40h | P0 |

---

### 3. 偿还计划

**目的**: 系统化偿还债务

#### 偿还策略

| 策略 | 描述 | 适用场景 |
|------|------|---------|
| **童子军规则** | 每次修改都改进 | 日常开发 |
| **专项重构** | 专门迭代偿还 | 高优先级债务 |
| **渐进重构** | 小步逐步改进 | 大债务 |

#### 偿还计划模板

```markdown
## 技术债务偿还计划

### Q1 计划
| 债务 | 预计工时 | 业务影响 | 状态 |
|------|---------|---------|------|
| [债务 1] | 20h | 低 | 计划中 |
| [债务 2] | 40h | 高 | 进行中 |

### 资源分配
- 总研发时间：100%
- 新功能：80%
- 债务偿还：20%
```
