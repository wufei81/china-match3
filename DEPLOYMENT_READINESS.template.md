### 回滚触发条件

出现以下任一情况立即回滚：

- [ ] P0 故障（核心功能不可用）
- [ ] 数据损坏/丢失
- [ ] 安全漏洞暴露
- [ ] 性能下降 >50%
- [ ] 错误率 >5%

### 回滚流程

```bash
# 1. 通知团队
echo "ROLLBACK INITIATED" in Slack

# 2. 停止流量 (如需要)
kubectl scale deployment ___ --replicas=0

# 3. 回滚到上一版本
kubectl rollout undo deployment/___

# 4. 验证回滚
curl https://___/health

# 5. 通知团队
echo "ROLLBACK COMPLETE" in Slack
```
