## 🎯 讨论议题

### 议题 1: 有限并行如何落地？
- PM 写 PRD 时，Arch 可以提前做什么？
- Dev 开发时，QA 如何同步准备测试？
- Ops 如何提前介入准备部署？

### 议题 2: 变更通知机制
- API 变更如何及时通知 QA 和前端？
- 设计变更如何通知开发和测试？
- 需求变更如何通知所有相关方？

### 议题 3: 质量门禁左移
- 开发前评审需要检查什么？
- 开发中自检查的频率和内容？
- 交付前自验证的标准？

### 议题 4: STATE.md 使用规范
- 更新频率：每小时？每任务完成？
- 更新内容：进度、阻塞、变更？
- 通知机制：自动@相关方？
