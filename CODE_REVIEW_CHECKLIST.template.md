### 1️⃣ 代码质量检查

| 检查项        | 要求                 | 检查方法                                 | 状态 | 备注     |
| ------------- | -------------------- | ---------------------------------------- | ---- | -------- |
| 无语法错误    | 代码可编译/解释      | `npm run build` / `python -m py_compile` | ⬜   |          |
| 无 lint 错误  | 符合代码规范         | `npm run lint` / `flake8`                | ⬜   |          |
| 无类型错误    | TypeScript 严格模式  | `tsc --noEmit`                           | ⬜   |          |
| 无 TODO/FIXME | 无占位符             | `grep -r "TODO\|FIXME" src/`             | ⬜   |          |
| 无调试代码    | 无 console.log/print | `grep -r "console.log\|print(" src/`     | ⬜   | 日志除外 |
| 无注释代码    | 无大段注释代码       | 人工审查                                 | ⬜   |          |
| 命名规范      | 变量/函数命名清晰    | 人工审查                                 | ⬜   |          |

## 🔧 自动化检查工具

**推荐在 CI 中集成的自动化检查**:

```bash
# 1. 代码质量
npm run lint
npm run test --coverage
tsc --noEmit

# 2. 安全检查
grep -r "SECRET\|PASSWORD" src/ | grep -v ".env.example"
grep -r 'f"SELECT\|f"INSERT\|f"DROP' src/
grep -r "console.log" src/ | grep -v "//"

# 3. 构建验证
npm run build
docker build -t test .

# 4. 健康检查
curl http://localhost:3000/health
```
